# Session Summary — April 15, 2026

## What Was Done Today

### 1. QA Audit (testing-qa-agent)
Full code review across all backend and frontend files. Findings:

**Critical issues found:**
- Race condition in double-booking prevention (FOR UPDATE lock acquired too late)
- Refund flow ignored cancellation fee
- Cancelled bookings could still be paid
- Unknown addon types silently priced at ₱0
- SECRET_KEY had insecure hardcoded fallback
- Payment method accepted any string (no enum validation)
- Location IDs not validated before booking creation
- Reports endpoint accepted inverted date ranges

---

### 2. Backend Fixes (backend-agent)
All 8 issues fixed:

| Fix | File |
|-----|------|
| Removed SECRET_KEY default — app errors at startup if not set | `config.py` |
| Payment method validated with `Literal[...]` enum | `schemas/payment.py` |
| Location IDs checked with 404 if not found | `routes/bookings.py` |
| Unknown addon types raise HTTP 422 | `routes/bookings.py`, `services/booking_service.py` |
| Explicit 409 guard blocks payment on cancelled bookings | `routes/payments.py` |
| Refund deducts cancellation_fee, returns refund_amount | `routes/payments.py` |
| Reports reject to_date < from_date with HTTP 400 | `routes/reports.py` |
| FOR UPDATE lock acquired before any other read in booking transaction | `routes/bookings.py` |

Also fixed `datetime.utcnow()` deprecation warnings across all model files and services:
- `models/user.py`, `models/booking.py`, `models/vehicle.py`, `models/payment.py`, `models/review.py`
- `services/auth_service.py`, `routes/payments.py`
- Replaced with `datetime.now(UTC)`
